Le format sf3, mis au point par MuseScore, est en tout point similaire au format sf2 à la différence suivante : les échantillons sont stockés dans le format OGG (comparable au format MP3 mais open source) au lieu d'être stockés en données brutes. La conséquence est que le format sf3 est environ 10 fois plus léger que le format sf2 pour une qualité comparable.

Une soundfont exportée dans ce format n'aura pas pour objectif d'être modifiée par la suite car des compressions successives entraineraient une baisse de qualité du son. Ce format doit donc être vu comme un produit final et est très intéressant pour tous les utilisateurs finaux de soundfonts :

  • téléchargements facilités sur internet,
  • gain de mémoire dans les ordinateurs,
  • la combinaison midi + sf3 offre une solution légère et de qualité pour la lecture de bandes sonores (jeux vidéos, applications mobiles, ...).

Contrairement au format sf2pack qui répond à la même problématique, le format sf3 est entièrement open-source. Le code source est disponible et lui assure ainsi sa pérénité (ne faisons pas les mêmes erreurs qu'avec sfArk ou sfPack !).
